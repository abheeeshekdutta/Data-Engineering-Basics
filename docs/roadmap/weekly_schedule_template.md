# Weekly Schedule Template (8-12 Hours)

## Block Plan

- Block A (2h): docs and theory reading
- Block B (2-3h): implementation sprint
- Block C (2-3h): implementation sprint
- Block D (2h): tests and debugging
- Block E (1-2h): retrospective and planning

## Weekly Review Questions

1. What failed unexpectedly and why?
2. What assumptions were wrong?
3. Which tests should be added to prevent recurrence?
4. What is the highest-risk item next week?
